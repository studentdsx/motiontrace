# UI 与交互设计

## 1. 设计方向

产品气质定位为「地图仪表盘 + 私人运动手账」。界面以地图作为第一视觉，不做营销页；所有入口围绕记录、打卡、回看展开。整体视觉采用干净的浅色地图底、墨绿色操作色、珊瑚色打卡强调色和深灰文字，避免单一蓝绿运动 App 的常见套路。

## 2. 信息架构

- 首页：今日地图、记录开关、实时统计、打卡入口、今日打卡时间线。
- 历史：按天归档的轨迹摘要。
- 日期详情：当天地图回放、统计卡片、打卡照片。
- 设置：定位与隐私说明、本地数据清理。

## 3. 首页

### 3.1 页面目标

让用户一打开就能完成两个动作：开始记录、沿途打卡。

### 3.2 布局

- 顶部为状态栏：日期、记录状态、记录开关。
- 主区域为地图，占据首屏主体。
- 地图下方为统计条：距离、时长、点数、打卡数。
- 底部为操作区：打卡按钮、居中按钮。
- 下方展示今日打卡列表。

### 3.3 状态

- 未记录：地图居中到当前位置或默认城市，开关关闭。
- 记录中：开关打开，状态显示「记录中」，轨迹线实时增长。
- 无权限：弹窗说明原因，引导打开设置。
- 无数据：统计归零，打卡列表为空状态。

## 4. 打卡交互

1. 用户点击「打卡」。
2. 获取当前位置。
3. 调用相册或相机选择照片。
4. 底部弹层展示已选照片和备注输入框。
5. 用户保存后关闭弹层，地图增加标记。

打卡不强制必须在记录中，原因是用户可能临时补记某个地点。

## 5. 历史页

### 5.1 页面目标

快速扫一眼过去每天有没有记录，以及某天运动量大概多少。

### 5.2 列表项

- 日期和星期。
- 距离、时长、打卡数。
- 一条小型摘要文案，如「18 个轨迹点」。
- 点击进入详情。

## 6. 日期详情页

### 6.1 页面目标

让一天的路线像一页运动日记：上方是地图，下方是数字和照片时间线。

### 6.2 交互

- 地图展示当天完整轨迹和打卡标记。
- 点击照片可预览。
- 无打卡时展示轻量空状态。

## 7. 设置页

- 说明当前版本只使用前台定位。
- 说明数据保存在本机。
- 提供清空本地数据按钮。

## 8. 视觉细节

- 记录中使用墨绿色强调。
- 打卡使用珊瑚色强调。
- 卡片半径控制在 8px 内，适合工具型小程序。
- 数字使用更醒目的字号，中文标签克制。
- 底部弹层不遮住核心操作太久，保存或取消后立即回到地图。
