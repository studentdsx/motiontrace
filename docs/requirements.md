# 运动轨迹小程序需求文档

## 1. 产品定位

这是一款面向个人日常运动与出行记录的小程序。用户可以在散步、跑步、骑行、通勤或旅行时打开轨迹记录开关，小程序持续获取前台定位，在地图上形成当天轨迹；用户可以在途中某个地点打卡、上传照片并添加文字备注。历史数据按天归档，方便回看每日路线、距离、时长和打卡照片。

## 2. 目标用户

- 有日常步行、跑步、骑行记录习惯的用户。
- 想记录旅行、城市漫游、校园路线的人。
- 希望把「路线 + 地点 + 照片」组织成个人运动日记的人。

## 3. 核心价值

- 少操作：打开开关即可开始记录，结束时关闭开关。
- 有记忆点：路线不仅是线，还包含沿途照片和文字打卡。
- 易回看：按天组织，能快速找到某一天的轨迹。
- 隐私优先：MVP 阶段数据默认保存在本机，不上传云端。

## 4. MVP 范围

### 4.1 轨迹记录

- 用户在首页打开「记录轨迹」开关后，小程序请求定位授权。
- 授权成功后开始前台定位更新。
- 每次定位变化会记录经纬度、速度、精度、时间戳。
- 地图实时绘制当天轨迹线。
- 首页展示今日距离、记录时长、轨迹点数、打卡数。
- 关闭开关后停止定位更新。

### 4.2 按天归档

- 轨迹数据按自然日保存。
- 历史页按日期倒序展示每日摘要。
- 每日摘要包含距离、时长、轨迹点数、打卡数。
- 点击某一天进入详情页，查看当天轨迹地图和打卡列表。

### 4.3 位置打卡

- 用户在记录过程中或首页点击「打卡」。
- 小程序获取当前定位。
- 用户选择或拍摄照片，最多 3 张。
- 用户可以填写一段文字备注。
- 打卡点展示在地图上，并出现在当天时间线中。

### 4.4 本地数据

- MVP 使用 `wx.setStorageSync` 保存轨迹、打卡和照片本地路径。
- 用户可以在设置页清空本地数据。
- 暂不做账号、云同步、分享、多人协作。

## 5. 非目标范围

- 后台持续定位。
- 运动健康专业训练计划。
- 社交动态流。
- 云端备份和跨设备同步。
- 复杂地图路线纠偏。

## 6. 关键流程

### 6.1 首次记录

1. 用户进入首页。
2. 打开记录开关。
3. 小程序请求定位权限。
4. 权限通过后开始记录，并在地图上显示当前位置。
5. 用户运动过程中轨迹线持续增长。
6. 用户关闭开关，小程序停止记录。

### 6.2 打卡

1. 用户点击「打卡」。
2. 小程序获取当前位置。
3. 用户选择照片。
4. 用户输入备注。
5. 保存后，地图增加打卡标记，时间线增加打卡卡片。

### 6.3 历史回看

1. 用户进入历史页。
2. 按日期浏览轨迹摘要。
3. 点击某一天。
4. 查看当天完整轨迹、统计数据和所有打卡。

## 8. 权限与隐私

- 需要声明定位用途：用于记录运动轨迹、计算距离并支持沿途打卡。
- MVP 只在前台记录，不声明后台定位。
- 设置页说明数据保存在本机。
- 清空数据前进行二次确认。
- 后续如果引入云同步，需要增加隐私政策、用户导出和删除机制。

## 9. 边界与异常

- 用户拒绝定位：提示去设置页开启权限。
- 定位精度过低：忽略明显漂移点，避免轨迹乱跳。
- 没有轨迹点时打卡：先获取一次当前位置。
- 照片保存失败：允许保存文字和位置，但提示部分照片未保存。
- 数据为空：历史页显示空状态，引导用户开始记录。

## 10. 后续版本方向

- 云同步和账号体系。
- 后台持续定位。
- GPX/KML 导出。
- 轨迹分享海报。
- 运动类型分类：步行、跑步、骑行、通勤。
- 自动暂停：长时间静止时暂停计时。
- 路线热力图和月度统计。

## 11. Android 版补充

- Android 版已支持前台服务后台记录轨迹。
- Android 版已新增可选账号与云同步入口，服务端使用 Cloudflare Workers + D1 SQLite。
- 当前云同步上传轨迹、行程、打卡坐标和文字，不上传照片原图。
- 照片跨设备同步后续建议接入 Cloudflare R2。
